User-Defined Data Types in C++14/17

1. Structures (struct)

Structures group related variables under one name. Members are public by default.

struct Sensor {
    int id;
    float reading;
};
Sensor s1 = {1, 23.5};

Robotics Use: Represent sensor data, actuator states, etc.

2. Classes (class)

Classes are similar to structs but support encapsulation, inheritance, and polymorphism. Members are private by default.

class Motor {
private:
    int speed;
public:
    void setSpeed(int s) { speed = s; }
    int getSpeed() const { return speed; }
};

Robotics Use: Model motors, controllers, and robot behaviors.

3. Unions (union)

Unions allow storing different data types in the same memory location. Only one member can hold a value at a time.

union Data {
    int i;
    float f;
};

Robotics Use: Efficient memory use in embedded systems.

4. Enumerations (enum, enum class)

Enumerations define a set of named integral constants. enum class provides better type safety.

enum Direction { NORTH, SOUTH, EAST, WEST };
enum class Mode { AUTO, MANUAL };

Robotics Use: Define robot modes, directions, or states.

5. Typedef and using

Typedef and using create type aliases for readability and abstraction.

typedef unsigned int uint;
using SensorID = int;

Robotics Use: Simplify complex types like function pointers or template types.
