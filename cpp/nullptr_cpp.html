nullptr in C++11 and Later

Definition

nullptr is a keyword introduced in C++11 to represent a null pointer constant. It provides a type-safe way to indicate that a pointer does not point to any object or function.

Advantages Over NULL

  • Type Safety: nullptr has its own type (std::nullptr_t), preventing ambiguity in overload resolution.
  • Clear Semantics: It clearly indicates a null pointer, unlike NULL which is typically defined as 0.
  • Improved Overload Resolution: Functions overloaded for pointer and integer types can be resolved correctly using nullptr.

Usage Examples

// Example 1: Assigning nullptr
int* ptr = nullptr;

// Example 2: Function overload resolution
void func(int);
void func(char*);

func(nullptr); // Calls func(char*) due to type safety

Type Safety Benefits

Using NULL can lead to unexpected behavior in function overloads because it is just an integer constant. nullptr avoids this by being a distinct type.

Best Practices

  • Always use nullptr instead of NULL or 0 for null pointers in modern C++.
  • Prefer nullptr in all pointer-related code to ensure clarity and type safety.
  • Update legacy code to use nullptr where applicable.

Conclusion

nullptr is a safer and more expressive alternative to NULL in C++11 and later. It enhances code readability and prevents subtle bugs related to pointer and integer confusion.
