Enumerations in C++14/17

What is an Enumeration?

An enumeration (or enum) is a user-defined type consisting of a set of named integral constants. It improves code readability and maintainability by replacing magic numbers with meaningful names.

Syntax

enum Direction { NORTH, SOUTH, EAST, WEST };
Direction d = NORTH;

Scoped Enumerations: enum class

Introduced in C++11 and enhanced in C++14/17, enum class provides better type safety and avoids name clashes.

enum class Mode { AUTO, MANUAL };
Mode m = Mode::AUTO;

Differences: enum vs enum class

  • enum: Implicitly converts to int, names are in global scope.
  • enum class: No implicit conversion, names are scoped.

Use Cases in Robotics

  • Robot modes: enum class Mode { IDLE, ACTIVE, ERROR };
  • Sensor states: enum SensorState { OK, FAIL, TIMEOUT };
  • Direction control: enum Direction { FORWARD, BACKWARD, LEFT, RIGHT };

Best Practices

  • Prefer enum class for type safety.
  • Use meaningful names for enum values.
  • Use switch statements to handle enum logic.

Example

enum class RobotState { IDLE, MOVING, CHARGING };

void handleState(RobotState state) {
    switch(state) {
        case RobotState::IDLE: /* handle idle */ break;
        case RobotState::MOVING: /* handle movement */ break;
        case RobotState::CHARGING: /* handle charging */ break;
    }
}
