Derived Data Types in C++14/17

Derived data types are built from fundamental data types and are essential for building complex programs, especially in robotics where data structures and memory management are critical.

1. Arrays

An array is a collection of elements of the same type stored in contiguous memory locations.

int distances[5] = {10, 20, 30, 40, 50};

Use in Robotics: Storing sensor readings like distance measurements from ultrasonic sensors.

2. Pointers

Pointers store the memory address of another variable. They are powerful for dynamic memory management and interfacing with hardware.

int speed = 100;
int* ptr = &speed;
std::cout << *ptr; // Outputs 100

Use in Robotics: Accessing hardware registers or dynamically allocated sensor data.

3. References

A reference is an alias for another variable. It must be initialized when declared and cannot be changed to refer to another variable.

int velocity = 50;
int& ref = velocity;
ref = 60; // velocity is now 60

Use in Robotics: Efficient parameter passing in functions controlling actuators or processing sensor data.

4. Functions

Functions are blocks of code that perform specific tasks. They can return values and accept parameters.

double computeDistance(double speed, double time) {
    return speed * time;
}

Use in Robotics: Modularizing control logic, such as computing distance traveled or PID control routines.

Best Practices

  • Use std::array or std::vector for safer array handling.
  • Prefer smart pointers (std::unique_ptr, std::shared_ptr) over raw pointers.
  • Use references for efficient and safe parameter passing.
  • Keep functions short, focused, and reusable.

Mastering these derived types is crucial for writing efficient and maintainable robotics software in C++.
